## Overview

Layerswap Widget supports two main flows that can be configured using the `defaultTab` parameter:

- **Swap**: Cross-chain token transfers between different blockchain networks
- **CEX**: Direct deposits from centralized exchanges to your wallet

## Default Tab Configuration

You can control which flow is displayed initially by setting the `defaultTab` parameter in your widget configuration:

```typescript
<LayerswapProvider
  config={{
    apiKey: 'YOUR_API_KEY',
    version: 'mainnet',
    initialSettings: {
      defaultTab: 'swap' // or 'cex'
    }
  }}
  walletProviders={[EVMProvider]}
>
  <Swap />
</LayerswapProvider>
```

## Swap Flow

The swap flow enables users to transfer tokens between different blockchain networks. This is ideal for:

- Moving assets from one blockchain to another
- Bridge functionality
- Cross-chain token transfers
- Layer 2 to Layer 1 (or vice versa) transfers

### Swap Configuration Example

```typescript
import { LayerswapProvider, Swap } from '@layerswap/widget';
import { EVMProvider } from '@layerswap/wallet-evm';

export default function App() {
  return (
    <LayerswapProvider
      config={{
        apiKey: 'YOUR_API_KEY',
        version: 'mainnet',
        initialSettings: {
          defaultTab: 'swap',
          from: 'ETHEREUM_MAINNET',
          to: 'ARBITRUM_MAINNET',
          fromAsset: 'USDC',
          toAsset: 'USDC',
          destAddress: '0x1234567890abcdef1234567890abcdef12345678'
        }
      }}
      walletProviders={[EVMProvider]}
    >
      <Swap />
    </LayerswapProvider>
  );
}
```

### Swap Flow Features

<ResponseField name="from" type="string">
  Source blockchain network (e.g., 'ETHEREUM_MAINNET', 'POLYGON_MAINNET')
</ResponseField>

<ResponseField name="to" type="string">
  Destination blockchain network (e.g., 'ARBITRUM_MAINNET', 'OPTIMISM_MAINNET')
</ResponseField>

<ResponseField name="fromAsset" type="string">
  Token to transfer from the source network (e.g., 'USDC', 'ETH')
</ResponseField>

<ResponseField name="toAsset" type="string">
  Token to receive on the destination network (e.g., 'USDC', 'ETH')
</ResponseField>

<ResponseField name="destAddress" type="string">
  Destination wallet address where funds will be received
</ResponseField>

## CEX Flow

The CEX (Centralized Exchange) flow allows users to deposit tokens directly from their exchange accounts to a blockchain wallet. This is perfect for:

- Onboarding users from exchanges
- Simplifying the deposit process
- Reducing the number of steps for users
- Direct exchange-to-wallet transfers

### CEX Configuration Example

```typescript
import { LayerswapProvider, Swap } from '@layerswap/widget';
import { EVMProvider } from '@layerswap/wallet-evm';

export default function App() {
  return (
    <LayerswapProvider
      config={{
        apiKey: 'YOUR_API_KEY',
        version: 'mainnet',
        initialSettings: {
          defaultTab: 'cex',
          from: 'COINBASE',
          to: 'ARBITRUM_MAINNET',
          toAsset: 'USDC',
          destAddress: '0x1234567890abcdef1234567890abcdef12345678'
        }
      }}
      walletProviders={[EVMProvider]}
    >
      <Swap />
    </LayerswapProvider>
  );
}
```

### CEX Flow Features

<ResponseField name="from" type="string">
  Source centralized exchange (e.g., 'COINBASE', 'BINANCE', 'KRAKEN')
</ResponseField>

<ResponseField name="to" type="string">
  Destination blockchain network where funds will be deposited
</ResponseField>

<ResponseField name="toAsset" type="string">
  Token to receive on the destination network
</ResponseField>

<ResponseField name="destAddress" type="string">
  Destination wallet address for the deposit
</ResponseField>

<Note>
  When using the CEX flow, users will receive instructions for transferring funds from their exchange account to their blockchain wallet through Layerswap's managed deposit process.
</Note>

## Switching Between Tabs

Users can switch between swap and CEX flows directly in the widget UI, regardless of which tab is set as default. The `defaultTab` parameter only controls which flow is displayed when the widget first loads.

### Hiding the Tab Switcher

If you want to lock users into a specific flow and prevent them from switching tabs, you can hide the tab switcher using theme configuration:

```typescript
<LayerswapProvider
  config={{
    apiKey: 'YOUR_API_KEY',
    version: 'mainnet',
    initialSettings: {
      defaultTab: 'swap'
    },
    theme: {
      ...theme,
      header: {
        hideTabs: true
      }
    }
  }}
  walletProviders={[EVMProvider]}
>
  <Swap />
</LayerswapProvider>
```

<ResponseField name="header.hideTabs" type="boolean">
  Set to `true` to hide the tab switcher UI. Users will only see the flow specified in `defaultTab` and won't be able to switch between swap and CEX flows.
</ResponseField>

<Note>
  Hiding tabs is useful when you want to create a focused experience for a specific use case, such as a bridge-only interface or a CEX deposit-only flow.
</Note>
