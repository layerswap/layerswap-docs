## Overview

The Layerswap Widget offers extensive customization options, allowing you to tailor both the appearance and behavior to match your application's needs. Customize colors, layouts, header visibility, border radius, and more to create a seamless, branded user experience.

## Interactive Playground

<Card title="Try the Customization Playground" icon="palette" href="https://playground.layerswap.io">
  Experiment with all customization options in real-time. Test colors, layouts, styles, and configurations with live preview before implementing in your application.
</Card>

## Quick Start

Here's a basic example of customizing the widget with a custom theme:

```typescript
import { LayerswapProvider, Swap } from '@layerswap/widget';
import { EVMProvider } from '@layerswap/wallet-evm';

export default function App() {
  const customTheme = {
    primary: {
      DEFAULT: '99, 102, 241',
      '500': '99, 102, 241',
      'text': '255, 255, 255',
    },
    secondary: {
      DEFAULT: '30, 41, 59',
      '500': '30, 41, 59',
      'text': '148, 163, 184',
    }
  };

  return (
    <LayerswapProvider
      config={{
        apiKey: 'YOUR_API_KEY',
        version: 'mainnet',
        theme: customTheme,
      }}
      walletProviders={[EVMProvider]}
    >
      <Swap />
    </LayerswapProvider>
  );
}
```

## Generating Color Palettes

Creating consistent color shades (100-900) for your theme is easy with **Tailwind Shades**:

<Card title="Tailwind Shades Generator" icon="swatchbook" href="https://tailwindshades.app/">
  Generate beautiful color palettes with 10 shades from a single base color. Perfect for creating consistent primary and secondary color schemes.
</Card>

### How to Use Tailwind Shades

1. Visit [tailwindshades.app](https://tailwindshades.app/)
2. Enter your brand's primary color (hex, RGB, or HSL)
3. The tool generates 10 shades (100-900) plus a DEFAULT shade
4. Copy the generated colors and convert to Layerswap format

### Converting Tailwind Colors to Layerswap Format

Tailwind Shades provides colors in various formats. For Layerswap, use the **RGB values without the `rgb()` wrapper**:

**Tailwind Shades Output:**
```
rgb(99, 102, 241)
```

**Layerswap Format:**
```typescript
'99, 102, 241'
```

**Complete Example:**

```typescript
// From Tailwind Shades
const tailwindColor = {
  100: 'rgb(224, 231, 255)',
  200: 'rgb(199, 210, 254)',
  500: 'rgb(99, 102, 241)',
  // ... etc
}

// Convert to Layerswap format
const layerswapPrimary = {
  '100': '224, 231, 255',
  '200': '199, 210, 254',
  '500': '99, 102, 241',
  // ... etc
}
```

### Border Radius

<ResponseField name="borderRadius" type="'none' | 'small' | 'medium' | 'large' | 'extraLarge' | 'default'">
  Controls the roundness of corners for cards, buttons, and other UI elements.
</ResponseField>

**Available Options:**

- `'none'` - No rounded corners (0px)
- `'small'` - Slightly rounded (4px)
- `'medium'` - Moderately rounded (8px)
- `'large'` - More rounded (12px)
- `'extraLarge'` - Very rounded (16px)
- `'default'` - Widget's default border radius

```typescript
borderRadius: 'medium'
```

### Header Customization

<ResponseField name="header" type="object">
  Configuration for widget header visibility options.
</ResponseField>

<ResponseField name="header.hideMenu" type="boolean">
  Hide the menu button in the widget header.
</ResponseField>

<ResponseField name="header.hideTabs" type="boolean">
  Hide the tab switcher (swap/cex) in the widget header. Useful when you want users to stay in a specific flow.
</ResponseField>

<ResponseField name="header.hideWallets" type="boolean">
  Hide the wallet connection display in the widget header.
</ResponseField>

```typescript
header: {
  hideMenu: true,
  hideTabs: true,
  hideWallets: true,
}
```

### Card Background Style

<ResponseField name="cardBackgroundStyle" type="React.CSSProperties">
  Custom CSS styles for card backgrounds. Useful for creating transparent or custom-styled cards.
</ResponseField>

```typescript
cardBackgroundStyle: {
  backgroundColor: 'transparent',
  backdropFilter: 'blur(10px)',
}
```

### Hide Powered By

<ResponseField name="hidePoweredBy" type="boolean">
  Remove the "Powered by Layerswap" branding from the widget footer.
</ResponseField>

```typescript
hidePoweredBy: true
```

## Best Practices

### Color Format Requirements

<Warning>
  **Important**: All colors must be in RGB format **without** the `rgb()` wrapper. Use `'99, 102, 241'` instead of `'rgb(99, 102, 241)'`.
</Warning>

The widget internally handles the color formatting, so you only need to provide the numeric RGB values as strings.

### Using Tailwind Shades for Consistency

For the best results, use [Tailwind Shades](https://tailwindshades.app/) to generate your color palettes:

1. **Primary Color**: Choose your brand's main color
2. **Secondary Color**: Select a complementary neutral color (grays work well)
3. **Generate Shades**: Let the tool create consistent 100-900 shades
4. **Convert Format**: Remove `rgb()` wrapper and use values as strings

This ensures:
- Consistent color progression
- Proper contrast ratios between shades
- Professional-looking color schemes

### Test in the Playground First

<Card title="Test Your Theme" icon="flask" href="https://playground.layerswap.io">
  Before implementing your theme in production, test it in the interactive playground to see how it looks with real widget components.
</Card>

**Benefits of testing in playground:**
- See changes in real-time
- Test with different network/asset combinations
- Verify accessibility and readability
- Export working configuration

### Accessibility Considerations

When customizing your theme, ensure:

- **Sufficient Contrast**: Text should have at least 4.5:1 contrast ratio with backgrounds
- **Button Visibility**: Primary buttons should be clearly distinguishable
- **Error States**: Error colors should be clearly visible and different from success colors
- **Text Readability**: Ensure `text` properties provide good contrast on their respective backgrounds

### Color Palette Tips

**Primary Colors:**
- Use your brand's primary color as the base (500)
- Ensure button text is readable on primary backgrounds

**Secondary Colors:**
- Typically use neutral grays
- These affect most of the widget's surface area

**Status Colors:**
- Keep semantic meaning (red for errors, green for success, yellow/orange for warnings)
- Ensure foreground colors are readable on their backgrounds

## Complete Integration Example

Here's a complete example showing widget customization with theme, initial settings, and configuration:

```typescript
import { LayerswapProvider, Swap } from '@layerswap/widget';
import { EVMProvider } from '@layerswap/wallet-evm';
import { customTheme } from './theme';

export default function App() {
  return (
    <LayerswapProvider
      config={{
        apiKey: 'YOUR_API_KEY',
        version: 'mainnet',
        theme: customTheme,
        initialSettings: {
          defaultTab: 'swap',
          to: 'ARBITRUM_MAINNET',
          toAsset: 'USDC',
        }
      }}
      walletProviders={[EVMProvider]}
    >
      <Swap />
    </LayerswapProvider>
  );
}
```
